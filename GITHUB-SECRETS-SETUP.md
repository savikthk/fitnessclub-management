# Настройка GitHub Secrets для CI/CD

Для работы CI/CD pipeline необходимо настроить секреты в GitHub репозитории.

## Шаги настройки:

1. Перейдите в ваш репозиторий на GitHub
2. Нажмите на вкладку **Settings**
3. В левом меню выберите **Secrets and variables** → **Actions**
4. Нажмите **New repository secret**

## Необходимые секреты:

### KEYSTORE_PASSWORD

**Name:** `KEYSTORE_PASSWORD`  
**Value:** Пароль для keystore (используйте тот же, что при генерации сертификатов)

**Пример значения:** `mySecurePassword123!`

**Важно:** 
- Используйте надёжный пароль
- Не используйте пароль по умолчанию `changeit` в продакшене
- Этот пароль будет использоваться для генерации keystore в CI/CD pipeline

## Проверка настройки:

После добавления секрета, при следующем push в ветку `main` или `master`, GitHub Actions автоматически:

1. ✅ Сгенерирует цепочку сертификатов
2. ✅ Скомпилирует проект
3. ✅ Запустит тесты
4. ✅ Создаст JAR артефакт
5. ✅ Загрузит артефакты

Проверить статус можно во вкладке **Actions** вашего репозитория.
